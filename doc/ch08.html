Chapter 8. Unflagged Options

UnflaggedOptions are values whose meanings are indicated by their position on the command line relative to other UnflaggedOptions (see example in Chapter 2). JSAP expects UnflaggedOptions on the command line in the order in which they were registered with the JSAP.

"Greedy" UnflaggedOptions (set via UnflaggedOption.setGreedy(boolean)) consume all remaining unflagged values from the command line as multiple values. Obviously, your JSAP can have only one greedy UnflaggedOption, and it must be the last UnflaggedOption registered with the JSAP.
